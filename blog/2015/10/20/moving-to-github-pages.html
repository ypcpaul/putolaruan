Moving to Github Pages

Oct 20, 2015
1 minute read

After hosting this website of mine on a VPS for a year, I’ve decided to move it to Github Pages. For those who are not familiar with it, Github Pages allows you to serve either your personal, organization or your Github project’s web pages through github itself just by having a repository on your github accounts with its repo name matching a pattern. For personal websites like mine, the default URL would be githubaccountname.github.com. You can then configure your DNS settings on your domain provider to point to github (instructions here).

Why move it? For the past year, I was only using Amazon’s AWS free trial. After that expired, I tried looking into other VPS providers (DO, Linode, Vultr, etc.). I almost created an instance on Vultr, but I can’t justify the expense since I have no plans on using it for anything other than a website. Lastly, Github Pages uses Jekyll already, which is perfect since I already use it as my blogging platform!

putolaruan.com will stay here for now, until I find a reason to setup my own VPS again, or if I decide to use my Github Pages solely for my open source works (that is, if I get anything done at all!).

Enjoy life :)
